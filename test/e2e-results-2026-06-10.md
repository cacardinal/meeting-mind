## 3. E2E — Tactiq Scraper Harness (`test/tactiq-scraper.html`)

**What the harness does:** Simulates Tactiq's transcript panel DOM (tab bar, scrollable `#transcript-container`, Tailwind-style entry markup with speaker in `text-[#6dcc6d]` div and text in `flex-1 break-words` div, plus a `tactiq-rtc` meta tag for detection). An inline copy of the scraper logic runs a MutationObserver (`childList`, `subtree`, `characterData`) over the container, parses entries via `class*=` attribute selectors, dedupes by index+text, and accumulates `segments` (exposed at `window.__TEST_STATE`). Three buttons drive the scenarios: Add Entry, Update Last Entry (in-place), Add Multi-Speaker Burst.

**Method note:** Playwright MCP was unavailable (browser profile `~/.playwright-chrome` locked by another active Chrome session; not killed to avoid disrupting that workflow). Fell back to a node + jsdom (27.4.0, from project devDependencies) check that loads the harness HTML, executes its inline script (with `type="module"` stripped — jsdom does not execute module scripts; the script has no imports so semantics are unchanged), dispatches button clicks, and asserts on the page status text and `window.__TEST_STATE.segments`.
